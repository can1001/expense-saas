# 지출결의서 복제 기능 테스트 계획

## 개요

**대상 기능**: 지출결의서 복제 (`POST /api/expenses/[id]/duplicate`)
**관련 커밋**: `e684d5d feat: 지출결의서 복제 기능 추가`
**테스트 유형**: 수동 테스트 (E2E)

---

## 의존성 그래프

```
[사용자 로그인]
       │
       ▼
[지출결의서 존재] ──────────────────┐
       │                           │
       ▼                           ▼
[상세 페이지 복제 버튼]     [목록 카드 스와이프 복제]
       │                           │
       └───────────┬───────────────┘
                   ▼
         [POST /api/.../duplicate]
                   │
                   ▼
         [새 지출결의서 생성]
                   │
                   ▼
         [편집 페이지 이동]
```

---

## 테스트 단계

### Phase 1: 사전 조건 확인
- 테스트용 계정 로그인
- 복제할 지출결의서 확보 (본인 작성)
- 다양한 상태의 지출결의서 준비

### Phase 2: 정상 케이스 테스트
- 상세 페이지에서 복제
- 목록 페이지에서 복제 (모바일 스와이프)
- 복제된 데이터 정확성 검증

### Phase 3: 에러 케이스 테스트
- 권한 없는 복제 시도
- 인증되지 않은 상태에서 복제

### Phase 4: 엣지 케이스 테스트
- 첨부파일 없는 지출결의서
- 항목이 최대(10개)인 지출결의서
- 간편 지출결의서(v4.1.4) 복제

---

## 검증 기준

### 데이터 복제 검증
| 필드 | 예상 결과 |
|------|----------|
| committee | 원본과 동일 |
| department | 원본과 동일 |
| items (전체) | 원본과 동일 |
| applicantName | 원본과 동일 |
| bankName/accountNumber/accountHolder | 원본과 동일 |
| attachments | 원본 URL 참조 |
| requestDate | 오늘 날짜 |
| status | DRAFT |
| applicantSignature* | null (제외) |
| expenseDate | null |

### 동작 검증
- 복제 후 3초 이내 편집 페이지 이동
- alert 메시지 "지출결의서가 복제되었습니다." 표시
- 새 ID로 지출결의서 생성 확인
