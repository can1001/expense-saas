# 지출결의서 복제 기능 테스트 태스크

## Phase 1: 사전 조건 확인

### Task 1.1: 테스트 환경 준비
- [ ] dev 서버 실행 (`npm run dev`)
- [ ] 테스트 계정으로 로그인
- **검증**: 로그인 후 `/expenses` 페이지 접근 가능

### Task 1.2: 테스트 데이터 준비
- [ ] 본인 작성 지출결의서 확인 (최소 1건)
- [ ] 다양한 상태의 지출결의서 확보:
  - [ ] DRAFT (임시저장)
  - [ ] APPROVED_FINAL (최종승인)
  - [ ] REJECTED (반려) - 있다면
- **검증**: 각 상태별 지출결의서 ID 기록

## Phase 2: 정상 케이스 테스트

### Task 2.1: 상세 페이지 복제 (데스크톱)
- [ ] `/expenses/[id]` 접속
- [ ] 상단 "복제" 버튼 클릭
- [ ] alert 메시지 확인: "지출결의서가 복제되었습니다."
- [ ] 편집 페이지로 이동 확인 (`/expenses/[newId]/edit`)
- **검증**: URL에 새로운 ID 포함

### Task 2.2: 상세 페이지 복제 (모바일)
- [ ] 모바일 뷰포트로 전환 (개발자 도구)
- [ ] `/expenses/[id]` 접속
- [ ] 하단 복제 버튼 클릭
- [ ] 편집 페이지로 이동 확인
- **검증**: 모바일 UI에서 정상 동작

### Task 2.3: 목록 카드 스와이프 복제
- [ ] `/expenses` 목록 페이지 접속 (모바일 뷰)
- [ ] 지출결의서 카드 왼쪽 스와이프
- [ ] "복제" 버튼 표시 확인
- [ ] 복제 버튼 클릭
- [ ] 편집 페이지로 이동 확인
- **검증**: 스와이프 메뉴에 복제 옵션 존재

### Task 2.4: 복제된 데이터 검증
- [ ] 복제된 지출결의서 편집 페이지에서 확인:
  - [ ] 위원회/사역팀: 원본과 동일
  - [ ] 세부 항목 (항/목/세목, 적요, 금액): 원본과 동일
  - [ ] 청구인 이름/직책: 원본과 동일
  - [ ] 은행 정보: 원본과 동일
  - [ ] 첨부파일: 원본과 동일한 이미지 표시
  - [ ] 청구일자: 오늘 날짜
  - [ ] 서명: 없음 (빈 상태)
- **검증**: 모든 필드 값 일치 확인

### Task 2.5: 다양한 상태에서 복제
- [ ] DRAFT 상태 지출결의서 복제
- [ ] APPROVED_FINAL 상태 지출결의서 복제
- [ ] REJECTED 상태 지출결의서 복제 (있다면)
- **검증**: 모든 상태에서 복제 가능, 결과는 항상 DRAFT

## Phase 3: 에러 케이스 테스트

### Task 3.1: 타인 지출결의서 복제 시도
- [ ] 다른 사용자가 작성한 지출결의서 ID 확인
- [ ] API 직접 호출: `POST /api/expenses/[타인ID]/duplicate`
- [ ] 403 에러 응답 확인
- [ ] 에러 메시지: "본인이 작성한 지출결의서만 복제할 수 있습니다."
- **검증**: 권한 검증 정상 동작

### Task 3.2: 로그아웃 상태에서 복제 시도
- [ ] 로그아웃
- [ ] API 직접 호출: `POST /api/expenses/[id]/duplicate`
- [ ] 401 에러 응답 확인
- **검증**: 인증 검증 정상 동작

### Task 3.3: 존재하지 않는 ID로 복제 시도
- [ ] API 직접 호출: `POST /api/expenses/invalid-id-12345/duplicate`
- [ ] 404 에러 응답 확인
- **검증**: 존재하지 않는 리소스 처리 정상
