# 지출결의서 결재선 규칙 (Approval Line Rules)

## 1. 결재선 자동 산출 규칙

### 1.1 금액 기반 결재 단계 (교회 조직)
| 금액 범위 | 결재 단계 | 설명 |
|---------|---------|------|
| ~50만원 | 2단계 | 팀장 → 회계 |
| 50만원~300만원 | 3단계 | 팀장 → 회계 → 재정팀장 |
| 300만원~ | 3단계 + 특별승인 | 팀장 → 회계 → 재정팀장 (고액 승인) |

### 1.2 부서/위원회 기반 결재자 매핑 (교회 조직)
```
위원회(committee) → 부서(department) → 결재자 계층
```

**기본 결재 계층:**
- **1차 결재**: 해당 부서 팀장 (department manager) - 예산 집행 타당성 검토
- **2차 결재**: 회계 (accountant) - 금액 검토 및 회계 처리 확인
- **3차 결재**: 재정팀장 (finance manager) - 최종 승인 및 지출 확정

### 1.3 예산 항목별 특수 규칙 (교회 조직)
특정 예산 항목(category/subcategory)은 추가 결재자가 필요할 수 있음:
- **건축/시설비**: 시설팀장 검토 추가
- **선교/헌금**: 선교팀장 검토 추가
- **특별사업비**: 담당 위원회 위원장 검토 추가
- **300만원 이상 고액**: 담임목사 승인 필요 (선택 사항)

## 2. 결재선 수정 권한

### 2.1 작성자(기안자) 권한
작성자는 **제출 전** 다음 권한을 가짐:
- ✅ 결재자 순서 변경 (단, 최종 승인자는 고정)
- ✅ 동일 레벨 결재자 교체 (예: 부재중 → 대리자)
- ✅ 결재 단계 추가 (선택적 참조자 추가)
- ❌ 최종 승인자 제거/변경 불가
- ❌ 필수 결재자 제거 불가

### 2.2 결재선 고정 (스냅샷)
- **제출 시점**: 결재선이 JSON 스냅샷으로 고정됨
- **제출 후**: 결재선 변경 불가 (반려 후 재제출만 가능)
- **감사 추적**: 모든 결재선 변경 이력 기록

## 3. 결재 프로세스

### 3.1 상태 전이 (State Machine)
```
DRAFT (작성중)
  ↓ [제출]
PENDING (결재 대기)
  ↓ [1차 승인]
APPROVED_STEP_1 (1차 승인 완료)
  ↓ [2차 승인]
APPROVED_STEP_2 (2차 승인 완료)
  ↓ ...
APPROVED_FINAL (최종 승인)

또는

PENDING → REJECTED (반려)
  ↓ [재제출]
PENDING (다시 결재 대기)
```

### 3.2 결재 액션
| 액션 | 설명 | 권한자 | 결과 |
|-----|------|--------|------|
| SUBMIT | 제출 | 작성자 | DRAFT → PENDING |
| APPROVE | 승인 | 현재 결재자 | 다음 단계로 진행 |
| REJECT | 반려 | 결재자 | PENDING → REJECTED |
| RESUBMIT | 재제출 | 작성자 | REJECTED → PENDING |
| WITHDRAW | 회수 | 작성자 | PENDING → DRAFT |

### 3.3 순차 승인 규칙
- **순차 처리**: 1차 → 2차 → 3차... 순서대로만 승인 가능
- **건너뛰기 불가**: 중간 결재자를 건너뛸 수 없음
- **병렬 승인**: 동일 단계에 여러 결재자가 있으면 모두 승인 필요 (AND 조건)

## 4. 보안 및 무결성

### 4.1 자기결재 방지
- ❌ 작성자는 본인의 결재선에 포함될 수 없음
- ❌ 결재자는 본인이 작성한 지출결의서를 승인할 수 없음
- ⚠️ 예외: 재정팀장이 작성한 경우 → 담임목사 승인 또는 당회 사후 보고

### 4.2 권한 상승 방지
- ❌ 작성자가 상위 결재자를 임의로 제거 불가
- ❌ 금액 변경 시 결재선 재산출 필요
- ✅ 금액 증가 → 결재선 자동 업그레이드 (단, 제출 전만)
- ✅ 금액 감소 → 결재선 유지 또는 다운그레이드 선택

### 4.3 제출 후 수정 정책
**제출 후 지출결의서 수정이 필요한 경우:**

| 변경 항목 | 정책 | 처리 방법 |
|----------|------|----------|
| 적요/설명 수정 | ❌ 불가 | 반려 후 재제출 |
| 금액 변경 | ❌ 불가 | 반려 후 재작성 |
| 예산 항목 변경 | ❌ 불가 | 반려 후 재작성 |
| 은행 계좌 변경 | ✅ 가능 (제한적) | 최종 승인 전만 수정 가능 |

**기본 원칙**: 제출 후 핵심 값 변경 불가 → 반려 후 재제출

### 4.4 대결/대리 결재
- **대결 (代決)**: 결재권자 부재 시 상위자가 대신 결재
- **위임 (委任)**: 결재권자가 다른 사람에게 권한 위임
- **대리 (代理)**: 결재권자 지정 없이 시스템 자동 처리

구현 방법:
- `ApprovalStep.delegatedTo` 필드로 대리 결재자 지정
- `ApprovalLog`에 대리 사유 기록

## 5. 감사 로그 (Audit Trail)

### 5.1 기록 대상
모든 결재 액션은 감사 로그에 기록:
- ✅ 누가 (userId/userName)
- ✅ 언제 (timestamp)
- ✅ 무엇을 (action: APPROVE/REJECT/SUBMIT...)
- ✅ 왜 (comment/reason)
- ✅ 변경 전/후 상태 (previousState/newState)

### 5.2 로그 보관
- 영구 보관 (soft delete 없음)
- 변조 방지 (immutable log)
- IP 주소, User Agent 기록
- 결재선 변경 전/후 스냅샷 저장

## 6. 예외 시나리오

### 6.1 결재자 퇴사/부재
- **퇴사**: 자동으로 후임자에게 재배정
- **장기 부재**: 대리자 지정 시스템 활용
- **단기 부재**: 결재 대기 또는 상위자 대결

### 6.2 긴급 지출
- **긴급 플래그**: `isUrgent: true`
- **알림 강화**: 결재자에게 즉시 알림
- **시간 제한**: 24시간 내 승인 필요 (선택적)

### 6.3 소액 자동 승인
- 10만원 미만: 팀장 승인만으로 자동 최종 승인 (선택 사항)
- 단, 특정 예산 항목(헌금, 선교비 등)은 예외로 전체 결재 필요
