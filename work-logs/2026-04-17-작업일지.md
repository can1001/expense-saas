## 지출결의서 상세 페이지 뒤로가기 시 검색 조건 유지

### 문제 상황
- 목록 페이지에서 검색/필터 조건 설정 후 상세 페이지로 이동
- 상세 페이지에서 뒤로가기 버튼 클릭 시 `/expenses`로 하드코딩되어 있어 검색 조건이 모두 초기화됨

### 해결 방안
sessionStorage를 활용해 검색 URL을 저장하고 복원하는 방식 사용

### 주요 작업 내용

1. **목록 페이지 URL 동기화** (`app/expenses/page.tsx`)
   - `useSearchParams` 훅 사용을 위해 `Suspense` 래퍼 컴포넌트 추가
   - URL 파라미터에서 초기값 읽기 (검색어, 정렬, 필터 등)
   - 필터/검색 상태 변경 시 URL 자동 업데이트 (`window.history.replaceState`)
   - `handleRowClick`에서 상세 페이지 이동 전 현재 URL을 sessionStorage에 저장

2. **ExpenseCard 컴포넌트 수정** (`components/ExpenseCard.tsx`)
   - `handleViewDetail`에서 sessionStorage에 현재 URL 저장

3. **상세 페이지 뒤로가기 수정** (`app/expenses/[id]/page.tsx`)
   - `returnUrl` state 추가 (기본값: `/expenses`)
   - `useEffect`로 sessionStorage에서 저장된 URL 복원
   - `handleBack` 함수 생성 (sessionStorage 정리 후 저장된 URL로 이동)
   - 4개의 뒤로가기 버튼 모두 `handleBack` 사용하도록 수정

### 동작 방식
1. 목록 페이지에서 필터/검색 설정 → URL에 쿼리 파라미터 반영 (예: `/expenses?q=홍길동&committee=선교위원회`)
2. 지출결의서 클릭 → sessionStorage에 현재 URL 저장 후 상세 페이지로 이동
3. 상세 페이지에서 뒤로가기 → sessionStorage에서 URL 읽어서 해당 URL로 복귀
4. 브라우저 새로고침 → URL 파라미터에서 검색 조건 복원
