### 2. 청구인 서명 누락 버그 수정

**커밋:** `9c3ee45` fix: 신규 생성 + 제출 시 청구인 서명 누락 버그 수정

**문제:**
- 결재 승인 시 "청구인 서명이 없는 지출결의서입니다" 에러 발생
- 신규 생성 + 제출 경로에서 서명 없이 PENDING 상태로 생성됨

**원인:**
- `POST /api/expenses`에서 `status: 'PENDING'`으로 직접 제출 시 서명 검증 없음
- `applicantSignatureType`, `applicantSignatureData` 필드가 저장되지 않음

**해결:**
- 신규 생성 시 DRAFT로 먼저 생성 후 submit API 호출 (서명 포함)
- API에서 PENDING 직접 제출 차단 (안전장치)

**변경된 흐름:**
```
Before: 제출 클릭 → POST /api/expenses (PENDING) → 서명 없이 생성 ❌
After:  제출 클릭 → POST /api/expenses (DRAFT) → POST /submit (서명) ✓
```

**수정 파일:**
- `components/ExpenseForm.tsx`
- `app/api/expenses/route.ts`

### 3. 결재 목록 페이지네이션 버그 수정

**커밋:** `4563721` fix: 결재 목록 페이지네이션 버그 수정

**문제:**
- 결재함에 결재 대기 목록이 일부만 표시됨
- 15건이 있어야 하는데 9건만 표시

**원인:**
- DB에서 페이지네이션(limit 10) 적용 후 isMyTurn 필터링
- 10건 조회 → 필터 후 9건만 남아 표시
- 나머지 6건은 페이지네이션으로 인해 조회되지 않음

**해결:**
- 페이지네이션 순서 변경: 전체 조회 → isMyTurn 필터링 → 페이지네이션 적용

**변경된 흐름:**
```
Before: DB 조회(limit 10) → isMyTurn 필터 → 9건 표시 ❌
After:  DB 전체 조회 → isMyTurn 필터(15건) → 페이지네이션 → 10건 표시 ✓
```

**수정 파일:**
- `app/api/approvals/route.ts`
