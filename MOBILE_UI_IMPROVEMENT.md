# 모바일 UI 개선 방안

## 현재 상태 분석

현재 시스템은 **데스크톱/태블릿 중심**으로 설계되어 있으며, 모바일에서 다음과 같은 문제가 있습니다:

| 영역 | 현재 상태 | 문제점 |
|------|----------|--------|
| 네비게이션 | 수평 메뉴 고정 | 모바일 햄버거 메뉴 없음 |
| 어드민 사이드바 | `w-60` 고정 너비 | 모바일에서 화면 초과 |
| 테이블 목록 | 가로 스크롤 의존 | 터치 UX 불편 |
| 폼 입력 | 데스크톱 레이아웃 | 입력 필드 간격/크기 미최적화 |
| 터치 타겟 | `px-4 py-2` 버튼 | 44px 미만으로 터치하기 어려움 |
| 타이포그래피 | `text-5xl` 고정 | 작은 화면에서 오버플로우 |

---

## 개선 방안

### 1. 모바일 네비게이션 (햄버거 메뉴)

**현재**: 모든 메뉴가 수평으로 나열되어 작은 화면에서 공간 부족

**개선안**:
```
┌─────────────────────────────┐
│ ☰  지출결의서 관리     👤  │  <- 모바일 헤더
├─────────────────────────────┤
│ ┌─────────────────────────┐ │
│ │ 지출결의서 목록         │ │  <- 드로어 메뉴
│ │ 새 지출결의서           │ │
│ │ 예산 현황               │ │
│ │ ───────────────────     │ │
│ │ 관리자                  │ │
│ │ 로그아웃                │ │
│ └─────────────────────────┘ │
└─────────────────────────────┘
```

**구현 포인트**:
- `md:` 이상에서는 기존 수평 메뉴 유지
- `sm:` 이하에서는 햄버거 아이콘 + 슬라이드 드로어
- 드로어 외부 클릭 시 닫힘
- 애니메이션: `transform translate-x` 사용

### 2. 어드민 사이드바 반응형 개선

**현재**: 고정 너비 `w-60`로 모바일에서 레이아웃 깨짐

**개선안**:
```
모바일 (< 768px)           태블릿/데스크톱 (≥ 768px)
┌──────────────────┐       ┌──────────┬─────────────────┐
│ ☰ 어드민 대시보드│       │ 사이드바 │ 메인 콘텐츠     │
├──────────────────┤       │          │                 │
│                  │       │ 예산관리 │                 │
│   메인 콘텐츠    │       │ 사용자   │                 │
│   (전체 너비)    │       │ 설정     │                 │
│                  │       │          │                 │
└──────────────────┘       └──────────┴─────────────────┘

      ↓ 햄버거 클릭 시

┌──────────────────┐
│ ← 닫기           │
├──────────────────┤
│ 예산관리         │
│ 사용자 관리      │
│ 시스템 설정      │
└──────────────────┘
```

**구현 포인트**:
- 모바일: 오버레이 드로어 방식
- 태블릿: 축소된 아이콘 사이드바 (`w-16`)
- 데스크톱: 전체 사이드바 (`w-60`)

### 3. 테이블 → 카드 뷰 전환

**현재**: 8컬럼 테이블 + 가로 스크롤

**개선안 (모바일 카드 뷰)**:
```
┌─────────────────────────────┐
│ 2024-01-15                  │
│ ┌─────────────────────────┐ │
│ │ 홍길동                  │ │
│ │ 기획위원회 > 재정팀     │ │
│ │ ─────────────────────── │ │
│ │ 사무행정비              │ │
│ │ ₩ 150,000          승인 │ │
│ └─────────────────────────┘ │
├─────────────────────────────┤
│ 2024-01-14                  │
│ ┌─────────────────────────┐ │
│ │ 김철수                  │ │
│ │ 선교위원회 > 선교1팀    │ │
│ ...                         │
└─────────────────────────────┘
```

**구현 포인트**:
- `md:` 이상: 기존 테이블 유지
- `sm:` 이하: 카드 리스트 표시
- 핵심 정보만 카드에 표시 (날짜, 신청자, 금액, 상태)
- 카드 클릭 시 상세 페이지로 이동
- 스와이프로 빠른 액션 (선택적)

### 4. 폼 레이아웃 최적화

#### 4.1 예산 선택기 개선

**현재**: 5개 셀렉트박스가 세로로 나열

**개선안**:
```
┌─────────────────────────────┐
│ 예산 선택                   │
├─────────────────────────────┤
│ 위원회        [선택 ▼]      │  <- 전체 너비
├─────────────────────────────┤
│ 사역팀/부     [선택 ▼]      │
├─────────────────────────────┤
│ 예산(항)      [선택 ▼]      │
├─────────────────────────────┤
│ 예산(목)      [선택 ▼]      │
├─────────────────────────────┤
│ 예산(세목)    [선택 ▼]      │
└─────────────────────────────┘
```

**모바일 최적화 옵션**:
- **옵션 A**: 바텀시트 선택기 (iOS 스타일)
- **옵션 B**: 전체 화면 선택 모달
- **옵션 C**: 단계별 위자드 (한 번에 하나씩)

**추천**: 옵션 A (바텀시트) - 자연스러운 모바일 경험

```
┌─────────────────────────────┐
│                             │
│   위원회를 선택하세요       │
│ ┌─────────────────────────┐ │
│ │ ○ 기획위원회            │ │
│ │ ○ 선교위원회            │ │
│ │ ○ 교육위원회            │ │
│ │ ● 찬양위원회  ✓         │ │
│ └─────────────────────────┘ │
│         [ 확인 ]            │
└─────────────────────────────┘
      ↑ 바텀시트
```

### 5. 터치 타겟 크기 개선

**WCAG 권장사항**: 최소 44x44px

**현재 → 개선**:
| 요소 | 현재 | 개선 |
|------|------|------|
| 버튼 | `px-4 py-2` (~36px) | `px-4 py-3` (~44px) |
| 체크박스 | 기본 크기 | `w-6 h-6` + 터치 영역 확대 |
| 링크 | 텍스트만 | `py-2` 패딩 추가 |
| 드롭다운 | 기본 높이 | `min-h-[44px]` |

**구현**:
```css
/* 모바일 터치 타겟 */
@media (max-width: 768px) {
  .btn { @apply py-3; }
  .form-input { @apply min-h-[44px]; }
  .clickable-row { @apply min-h-[56px]; }
}
```

### 6. 반응형 타이포그래피

**현재**: 고정 크기 (`text-5xl`, `text-3xl` 등)

**개선안**:
```css
/* 제목 */
.heading-1 {
  @apply text-2xl sm:text-3xl md:text-4xl lg:text-5xl;
}

/* 부제목 */
.heading-2 {
  @apply text-xl sm:text-2xl md:text-3xl;
}

/* 본문 */
.body-text {
  @apply text-sm sm:text-base;
}
```

**예시 (홈페이지)**:
| 화면 크기 | 현재 | 개선 |
|----------|------|------|
| 320px (모바일) | `text-5xl` (48px) | `text-2xl` (24px) |
| 768px (태블릿) | `text-5xl` (48px) | `text-4xl` (36px) |
| 1024px+ (데스크톱) | `text-5xl` (48px) | `text-5xl` (48px) |

## 추가 권장사항

### 1. PWA (Progressive Web App) 지원
- `next-pwa` 패키지 추가
- 오프라인 지원
- 홈 화면 추가 기능
- 푸시 알림 (결재 알림)

### 2. 모바일 전용 기능
- 카메라로 영수증 촬영 → 첨부
- GPS 위치 기반 지출 장소 자동 입력
- 음성으로 적요 입력

### 3. 성능 최적화
- 이미지 lazy loading (이미 Next.js Image 사용 중)
- 무한 스크롤 (목록 페이지)
- 스켈레톤 로딩 UI
