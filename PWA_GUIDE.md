# PWA (Progressive Web App) 가이드

## PWA란?

PWA(Progressive Web App)는 웹 기술로 만든 앱으로, 웹사이트처럼 접근하면서도 네이티브 앱과 유사한 사용자 경험을 제공합니다.

### PWA의 장점

| 장점 | 설명 |
|------|------|
| **설치 가능** | 홈 화면에 아이콘으로 추가하여 앱처럼 실행 |
| **오프라인 지원** | 인터넷 없이도 캐시된 페이지 접근 가능 |
| **빠른 로딩** | 자주 사용하는 리소스를 캐싱하여 빠르게 로드 |
| **자동 업데이트** | 앱스토어 업데이트 없이 자동으로 최신 버전 유지 |
| **앱스토어 불필요** | 별도 심사 없이 바로 배포 및 사용 가능 |
| **저용량** | 네이티브 앱 대비 훨씬 적은 저장 공간 사용 |

### PWA vs 네이티브 앱

| 항목 | PWA | 네이티브 앱 |
|------|-----|------------|
| 설치 방법 | 브라우저에서 바로 설치 | 앱스토어에서 다운로드 |
| 업데이트 | 자동 (새로고침) | 수동 (앱스토어) |
| 저장 공간 | 수 MB | 수십~수백 MB |
| 하드웨어 접근 | 일부 (카메라, GPS 등) | 전체 |
| 오프라인 | 캐시 기반 제한적 지원 | 완전 지원 |

## 지출결의서 앱 설치 가이드

### iOS (iPhone, iPad)

1. **Safari** 브라우저로 사이트 접속
2. 하단의 **공유 버튼** (□↑) 탭
3. 스크롤하여 **"홈 화면에 추가"** 선택
4. 이름 확인 후 **"추가"** 탭

> **참고**: iOS에서는 Safari에서만 PWA 설치가 가능합니다. Chrome, Firefox 등에서는 설치 옵션이 나타나지 않습니다.

### Android

#### Chrome 브라우저
1. Chrome으로 사이트 접속
2. 주소창 옆 **메뉴 (⋮)** 탭
3. **"앱 설치"** 또는 **"홈 화면에 추가"** 선택
4. **"설치"** 확인

#### 자동 설치 배너
- 사이트를 여러 번 방문하면 자동으로 설치 배너가 표시됩니다.
- 배너의 **"설치"** 버튼을 탭하면 바로 설치됩니다.

### 데스크톱 (PC, Mac)

#### Chrome
1. 사이트 접속
2. 주소창 오른쪽의 **설치 아이콘** (⊕ 또는 모니터 아이콘) 클릭
3. **"설치"** 확인

#### Edge
1. 사이트 접속
2. 주소창 오른쪽의 **앱 설치 아이콘** 클릭
3. **"설치"** 확인

## 설치 후 사용법

### 앱 실행
- 홈 화면(모바일) 또는 앱 목록(데스크톱)에서 **"지출결의서"** 아이콘 탭/클릭
- 브라우저 주소창 없이 전체 화면으로 실행됩니다.

### 바로가기 기능
앱 아이콘을 길게 누르면 바로가기 메뉴가 나타납니다:
- **새 지출결의서**: 바로 새 결의서 작성 화면으로 이동
- **지출결의서 목록**: 결의서 목록 화면으로 이동

### 오프라인 사용
인터넷 연결이 끊어져도:
- 이전에 열어본 페이지는 캐시에서 확인 가능
- 지출결의서를 작성하고 임시저장 가능 (IndexedDB 저장)
- 오프라인에서 제출한 데이터는 온라인 복귀 시 자동 동기화
- 상단에 오프라인 상태 배너가 표시됨

## 캐싱 전략

지출결의서 앱은 다음과 같은 캐싱 전략을 사용합니다:

| 리소스 유형 | 전략 | 캐시 기간 | 설명 |
|------------|------|----------|------|
| Google 폰트 | Cache First | 1년 | 폰트는 거의 변경되지 않으므로 캐시 우선 |
| 정적 폰트 | Stale While Revalidate | 1주일 | 캐시 사용 후 백그라운드 업데이트 |
| 이미지 | Stale While Revalidate | 24시간 | 캐시 사용 후 백그라운드 업데이트 |
| JavaScript | Cache First | 24시간 | 빌드된 JS는 변경 없음 |
| CSS | Stale While Revalidate | 24시간 | 스타일 캐시 후 업데이트 |
| API 호출 | Network First | 24시간 | 네트워크 우선, 실패 시 캐시 (10초 타임아웃) |
| 기타 | Network First | 24시간 | 네트워크 우선, 실패 시 캐시 |

### 캐싱 전략 설명
- **Cache First**: 캐시에 있으면 캐시 사용, 없으면 네트워크
- **Network First**: 네트워크 먼저 시도, 실패하면 캐시 사용
- **Stale While Revalidate**: 캐시 즉시 반환 후 백그라운드에서 업데이트

## 알림 시스템

### 구현된 알림 방식

지출결의서 시스템은 다음 3가지 채널을 통해 알림을 발송합니다:

| 채널 | 설명 | 설정 필요 |
|------|------|----------|
| **SMS** | 문자 메시지 발송 | Notification Hub API |
| **카카오 알림톡** | 카카오톡 알림 메시지 | 템플릿 심사 필요 |
| **웹 푸시** | 브라우저/앱 알림 | VAPID 키 설정 |

### 알림 발송 시점

| 이벤트 | 수신자 | 알림 내용 |
|--------|--------|----------|
| **결재 제출** (SUBMIT) | 결재자 | 새로운 결재 요청이 있음을 알림 |
| **결재 승인** (APPROVE) | 신청자 | 결재가 승인되었음을 알림 |
| **결재 반려** (REJECT) | 신청자 | 결재가 반려되었음과 사유 알림 |
| **결재 회수** (WITHDRAW) | 대기중인 결재자 | 결재 요청이 회수되었음을 알림 |
| **지급 완료** (PAYMENT_COMPLETE) | 신청자 | 지급이 완료되었음을 알림 |

### 알림 채널

#### SMS
- 기본 알림 채널
- 짧은 문자 메시지로 발송
- 발신번호: [지출결의]

#### 카카오 알림톡
- 사전 심사된 템플릿 사용
- 미설정 시 SMS로 자동 대체 발송
- 링크 버튼 포함 가능

### 알림 설정 관리

사용자별로 알림 설정을 관리할 수 있습니다:

#### 채널별 설정
- SMS 수신 여부
- 카카오 알림톡 수신 여부
- 웹 푸시 알림 수신 여부

#### 이벤트별 설정
- 결재 제출 알림
- 결재 승인 알림
- 결재 반려 알림
- 지급 완료 알림

### 알림 로그

모든 알림 발송 내역은 데이터베이스에 기록됩니다:
- 수신자 정보 (이름, 전화번호)
- 발송 채널 (SMS/KAKAO/WEB_PUSH)
- 이벤트 유형
- 메시지 내용
- 발송 상태 (성공/실패)
- 오류 메시지 (실패 시)

## 관리자 설정

### 알림 서비스 활성화

환경 변수로 알림 서비스를 제어합니다:

```bash
# 알림 활성화 (기본값: true)
NOTIFICATION_ENABLED=true

# 알림 비활성화
NOTIFICATION_ENABLED=false
```

### 카카오 알림톡 설정

카카오 알림톡 사용을 위해 다음 환경 변수가 필요합니다:

```bash
# 카카오 알림톡 템플릿 코드 (이벤트별)
KAKAO_TEMPLATE_SUBMIT=템플릿코드1
KAKAO_TEMPLATE_APPROVE=템플릿코드2
KAKAO_TEMPLATE_REJECT=템플릿코드3
KAKAO_TEMPLATE_WITHDRAW=템플릿코드4
KAKAO_TEMPLATE_PAYMENT_COMPLETE=템플릿코드5
```

### 웹 푸시 알림 설정

웹 푸시 알림을 사용하려면 VAPID 키가 필요합니다:

```bash
# VAPID 키 생성 (최초 1회)
npx web-push generate-vapid-keys

# 환경 변수 설정
VAPID_PUBLIC_KEY="생성된_공개키"
VAPID_PRIVATE_KEY="생성된_비밀키"
VAPID_SUBJECT="mailto:admin@example.com"
```

#### VAPID 키 생성 방법

```bash
# 프로젝트 루트에서 실행
npx web-push generate-vapid-keys
```

출력 예시:
```
Public Key: BNxS...abc
Private Key: xyz...123
```

#### 웹 푸시 API 엔드포인트

| 엔드포인트 | 메서드 | 설명 |
|-----------|--------|------|
| `/api/push/vapid-public-key` | GET | VAPID 공개키 조회 |
| `/api/push/subscribe` | POST | 푸시 구독 등록 |
| `/api/push/unsubscribe` | POST | 푸시 구독 해제 |
| `/api/push/test` | POST | 테스트 알림 발송 |

## 오프라인 데이터 저장

### IndexedDB 기반 오프라인 저장

인터넷 연결이 없을 때도 지출결의서를 작성하고 저장할 수 있습니다.

#### 작동 방식

1. **오프라인 감지**: 네트워크 상태를 자동으로 감지합니다.
2. **로컬 저장**: 오프라인 시 데이터가 브라우저 IndexedDB에 저장됩니다.
3. **자동 동기화**: 온라인 복귀 시 자동으로 서버와 동기화됩니다.

#### 저장 가능한 데이터

- 지출결의서 초안 (임시저장)
- 제출 대기 중인 결의서
- 첨부파일 (최대 5MB)

#### 동기화 상태

| 상태 | 설명 |
|------|------|
| `draft` | 임시저장 (동기화 대상 아님) |
| `pending_sync` | 동기화 대기 중 |
| `syncing` | 동기화 진행 중 |
| `synced` | 동기화 완료 |
| `conflict` | 충돌 발생 (수동 해결 필요) |
| `failed` | 동기화 실패 |

#### 오프라인 배너

오프라인 상태가 되면 화면 상단에 배너가 표시됩니다:
- 현재 네트워크 상태
- 동기화 대기 중인 항목 수
- 수동 동기화 버튼

## 백그라운드 동기화

### Background Sync API

오프라인에서 제출한 데이터는 백그라운드에서 자동으로 동기화됩니다.

#### 지원 브라우저

| 브라우저 | Background Sync | 대체 방식 |
|----------|----------------|----------|
| Chrome | O | - |
| Edge | O | - |
| Firefox | X | 온라인 이벤트 기반 |
| Safari | X | 온라인 이벤트 기반 |

#### 동작 과정

1. **오프라인 제출**: 사용자가 오프라인에서 지출결의서 제출
2. **동기화 등록**: Service Worker에 동기화 태그 등록
3. **자동 재시도**: 온라인 복귀 시 자동으로 API 호출
4. **결과 알림**: 동기화 결과를 사용자에게 알림

#### 재시도 정책

- 최대 재시도 횟수: 3회
- 재시도 간격: 지수 백오프 (1분, 2분, 4분)
- 실패 시: 수동 재시도 버튼 제공

## 웹 푸시 알림 사용법

### 알림 권한 요청

1. 사이트 접속 시 알림 권한 요청 배너가 표시됩니다.
2. **"알림 받기"** 버튼을 클릭합니다.
3. 브라우저의 권한 요청 팝업에서 **"허용"**을 선택합니다.

### 알림 수신

권한을 허용하면 다음 이벤트 발생 시 푸시 알림을 받습니다:
- 새 결재 요청 도착
- 결재 승인/반려
- 지급 완료

### 알림 설정 변경

알림 설정 페이지에서 다음을 변경할 수 있습니다:
- 웹 푸시 알림 켜기/끄기
- 이벤트별 알림 설정

### 알림이 안 올 때

1. **브라우저 설정 확인**: 사이트 알림이 차단되어 있지 않은지 확인
2. **권한 재요청**: 알림 설정에서 권한을 다시 요청
3. **Service Worker 확인**: 개발자 도구 > Application > Service Workers

## 문제 해결

### 설치 옵션이 나타나지 않음
- **iOS**: Safari 브라우저를 사용하고 있는지 확인
- **Android/Chrome**: HTTPS 연결인지 확인 (localhost 제외)
- 이미 설치된 경우 설치 옵션이 나타나지 않음

### 오프라인에서 작동하지 않음
- 해당 페이지를 온라인에서 한 번 이상 방문해야 캐시됩니다
- 캐시 용량 제한으로 오래된 캐시가 삭제되었을 수 있습니다

### 앱이 업데이트되지 않음
- 앱을 완전히 종료 후 다시 실행
- 브라우저 캐시 삭제 후 재설치

### 알림이 오지 않음
- 휴대폰 번호가 올바르게 등록되어 있는지 확인
- 알림 설정에서 해당 이벤트가 활성화되어 있는지 확인
- 관리자에게 알림 서비스 활성화 여부 문의

## 기술 스택

- **PWA 프레임워크**: next-pwa
- **Service Worker**: Workbox + Custom Worker
- **캐싱**: CacheFirst, NetworkFirst, StaleWhileRevalidate
- **오프라인 저장**: Dexie.js (IndexedDB)
- **백그라운드 동기화**: Background Sync API, SyncManager
- **웹 푸시**: web-push (VAPID)
- **알림 서비스**: SMS, 카카오 알림톡, 웹 푸시
