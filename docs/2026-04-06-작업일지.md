- 회계(accountant), 행정간사(admin_assistant) 등의 역할이 연도별로 변경될 수 있음
- 기존: `User.role` 필드만 참조 → 연도별 역할 반영 불가
- 변경: `UserYearRole` 테이블 우선 참조

**변경 내용:**

1. **lib/auth.ts - getCurrentUser 함수**
   - 현재 연도의 UserYearRole 조회
   - YearRole이 있으면 해당 역할 사용, 없으면 User.role 사용
   ```typescript
   const yearRole = await prisma.userYearRole.findFirst({
     where: { userId, year: currentYear },
     orderBy: { role: 'asc' },
   });
   return {
     ...userInfo,
     role: yearRole?.role ?? userInfo.role,
   };
   ```

2. **lib/services/user-service.ts - findUsers 함수**
   - 역할 필터링 시 UserYearRole 테이블 조회
   - admin 역할만 User.role에서 조회 (영구 역할)
   - 그 외 역할은 UserYearRole에서 해당 연도 기준 조회

**영향:**
- 메뉴 권한 체크가 UserYearRole 기반으로 동작
- 회계/행정간사 등 연도별 역할자가 정상적으로 메뉴 접근 가능
- 사용자 관리 페이지 역할 필터링이 UserYearRole 기반으로 동작
