## 프로젝트 개요

### 목표
단일 테넌트 지출결의서 시스템을 멀티테넌시 SaaS 플랫폼으로 전환

### 핵심 요구사항
- 여러 조직(교회, 비영리단체, 학교, 기업)이 독립적으로 서비스 이용
- 서브도메인 기반 테넌트 식별 (`chungyeon.expense-saas.com`)
- 데이터 완전 격리 (Cross-tenant 접근 방지)
- 중앙 플랫폼 관리 (SuperAdmin)

### 기술 스택
- Next.js 16 (App Router)
- PostgreSQL (Neon)
- Prisma 7.0 (Extension 기반 자동 필터링)
- AsyncLocalStorage (요청 범위 컨텍스트)

## 작업 타임라인

### Phase 1: 스키마 확장 및 인프라

**데이터베이스 스키마**
- Tenant 모델 추가 (name, subdomain, customDomain, plan, orgType 등)
- SuperAdmin 모델 추가 (플랫폼 관리자)
- 모든 비즈니스 모델에 tenantId 필드 추가
- 기존 데이터 마이그레이션 스크립트 작성

**인프라 구성**
- AsyncLocalStorage 기반 테넌트 컨텍스트 (`lib/tenant-context.ts`)
- Prisma Extension 자동 필터링 (`lib/prisma-tenant-extension.ts`)
- 서브도메인 추출 미들웨어 (`middleware.ts`)
- 테넌트 조회 헬퍼 및 캐싱 (`lib/tenant.ts`)

### Phase 2: 인증 시스템 강화

**사용자 인증**
- JWT 토큰에 tenantId 포함
- withAuth 래퍼에 테넌트 컨텍스트 자동 설정
- 서브도메인-테넌트 일치 검증 추가
- 권한 기반 접근 제어 (withPermission)

**SuperAdmin 인증**
- 별도의 JWT 토큰 체계 (`super_admin_token`)
- withSuperAdmin 래퍼 구현
- 플랫폼 레벨 API 보호

### Phase 3: 플랫폼 관리 UI

**관리자 인터페이스**
- SuperAdmin 로그인 페이지 (`/platform/login`)
- 대시보드 (`/platform/dashboard`)
- 테넌트 목록 (`/platform/tenants`)
- 테넌트 상세/관리 (`/platform/tenants/[id]`)
- 테넌트 사용자 목록 (`/platform/tenants/[id]/users`)

**API 엔드포인트**
- 플랫폼 인증 API (`/api/platform/auth/*`)
- 테넌트 CRUD API (`/api/platform/tenants/*`)
- 테넌트 통계 API (`/api/platform/tenants/[id]/stats`)

### Phase 4: 테넌트 UI 연동

**테넌트 정보 표시**
- 로그인 페이지에 테넌트 로고/이름 표시
- 헤더에 테넌트 이름 표시
- 테넌트 정보 조회 API (`/api/tenant/info`)

**시드 데이터**
- SuperAdmin 시드 스크립트 (`prisma/seeds/super-admin-seed.ts`)
- 샘플 테넌트 시드 스크립트 (`prisma/seeds/tenant-seed.ts`)

## 구현 상세

### 1. 테넌트 컨텍스트 관리

```typescript
// lib/tenant-context.ts
const tenantStorage = new AsyncLocalStorage<TenantContext>();

export function withTenantAsync<T>(
  context: TenantContext,
  fn: () => Promise<T>
): Promise<T> {
  return tenantStorage.run(context, fn);
}

export function getTenantId(): string {
  const ctx = tenantStorage.getStore();
  if (!ctx?.tenantId) throw new Error('Tenant context not available');
  return ctx.tenantId;
}
```

### 2. Prisma 자동 필터링

```typescript
// lib/prisma-tenant-extension.ts
export function createTenantExtension() {
  return Prisma.defineExtension((client) =>
    client.$extends({
      query: {
        $allModels: {
          async findMany({ model, args, query }) {
            const tenantId = getTenantIdOptional();
            if (tenantId && TENANT_SCOPED_MODELS.includes(model)) {
              args.where = { ...args.where, tenantId };
            }
            return query(args);
          },
          // create, update, delete도 동일하게 처리
        },
      },
    })
  );
}
```

### 3. 서브도메인 검증

```typescript
// lib/auth/user.ts - withAuth 내부
const subdomain = request.headers.get('x-tenant-subdomain');
const requestedTenant = subdomain || request.headers.get('x-tenant-param');

if (requestedTenant) {
  const resolvedTenant = await findTenantBySubdomain(requestedTenant);

  if (!resolvedTenant) {
    return NextResponse.json(
      { error: '존재하지 않거나 비활성화된 조직입니다.' },
      { status: 404 }
    );
  }

  if (resolvedTenant.tenantId !== user.tenantId) {
    return NextResponse.json(
      { error: '이 조직에 대한 접근 권한이 없습니다.' },
      { status: 403 }
    );
  }
}
```

### 4. 미들웨어 서브도메인 추출

```typescript
// middleware.ts
function extractSubdomain(hostname: string): string | null {
  const parts = hostname.split('.');
  if (parts.length >= 3 && !RESERVED_SUBDOMAINS.includes(parts[0])) {
    return parts[0];
  }
  return null;
}
```
