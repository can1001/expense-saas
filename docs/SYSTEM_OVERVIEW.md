# 지출결의서 관리 시스템 개요

## 1. 핵심 사용 목적 (Why)

### 🎯 목적 한 줄 요약

> 지출결의 시스템은 **'지출을 처리하는 도구'** 가 아니라,
> **예산 편성 → 집행 → 통제 → 결산**까지 이어지는 **재정 흐름을 구조화하는 시스템**입니다.

---

### 재정 관리의 본질

```
┌─────────────────────────────────────────────────────────────┐
│                                                             │
│   1. 예산은 미리 계획되고                                    │
│                                                             │
│   2. 지출은 통제된 절차로 집행되며                           │
│                                                             │
│   3. 모든 집행 결과는 자동으로 결산 자산이 된다               │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

**핵심 철학:**

> **내년 결산을 편하게 하기 위해, 올해 지출을 체계적으로 관리하는 시스템**

---

### 두 가지 관점

#### 1. 재정 관리 관점

예산의 전체 생명주기를 관리합니다.

| 단계 | 설명 | 시스템 기능 |
|------|------|------------|
| **편성** | 연초에 예산을 계획 | 5단계 계층 구조, 연도별 예산 배정 |
| **집행** | 통제된 절차로 지출 | 결재 워크플로우, 담당자 승인 |
| **통제** | 실시간 모니터링 | 사용률 추적, 예산 초과 경고 |
| **결산** | 자동으로 데이터 축적 | 사용 금액 집계, 보고서 생성 |

#### 2. 은행 연동 관점

승인된 지출을 실제 자금으로 집행합니다.

| 단계 | 설명 | 시스템 기능 |
|------|------|------------|
| **지출 예정** | 결재 완료된 지출건 | 최종 승인 상태 관리 |
| **자금 집행** | 은행으로 이체 요청 | 엑셀 내보내기 → 은행 업로드 |
| **완료 추적** | 지급 상태 기록 | 감사 로그, 결산 데이터 확정 |

---

## 2. Big Picture ① : 예산 · 집행 · 결산을 하나의 흐름으로

### 기존 방식의 문제

```
┌─────────────┐    ┌─────────────┐    ┌─────────────┐
│   예산      │    │   지출      │    │   결산      │
│  (엑셀/문서) │    │ (수기 결의서)│    │ (연말 수작업)│
└─────────────┘    └─────────────┘    └─────────────┘
       ↓                 ↓                 ↓
    따로 관리          따로 처리          따로 정리
```

**문제점:**
- 예산: 엑셀 / 문서로 별도 관리
- 지출: 수기 결의서 + 개별 계좌 이체
- 결산: 연말에 다시 수작업 정리

> **"지출은 했는데, 예산과 결산은 따로 놀음"**

---

### 지출결의 시스템이 만드는 구조

```
┌─────────────────────────────────────────────────────────────┐
│                    하나의 통합된 흐름                         │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│   [예산 편성]                                                │
│        │                                                    │
│        ▼                                                    │
│   [지출 예정 등록 (결의)]  ← 예산 항목에 귀속                 │
│        │                                                    │
│        ▼                                                    │
│   [승인 프로세스]         ← 담당자 → 회계 → 재정팀장          │
│        │                                                    │
│        ▼                                                    │
│   [실제 지출]             ← 엑셀 → 은행 이체                 │
│        │                                                    │
│        ▼                                                    │
│   [자동 누적 · 분류]      ← 사용 금액 실시간 집계             │
│        │                                                    │
│        ▼                                                    │
│   [결산 데이터 완성]      ← 조회만 하면 끝                    │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

---

### 핵심 포인트

| 기존 | 시스템 |
|------|--------|
| 지출은 **사후 기록** | 지출은 **사전 계획** (결의 → 승인 → 집행) |
| 예산과 지출이 **분리** | 모든 지출은 **예산 항목에 귀속** |
| 결산은 **정리 작업** | 결산은 **조회 버튼** |

---

## 핵심 기능

### 1. 예산 관리

#### 1.1 예산 계층 구조 (5단계)

```
위원회 (Committee)
    └── 사역팀/부 (Department)
            └── 예산(항) (BudgetCategory)
                    └── 예산(목) (BudgetSubcategory)
                            └── 예산(세목) (BudgetDetail)
```

**예시:**
```
재정위원회
    └── 재정팀
            └── 사무행정비
                    └── 회의접대비
                            └── 아웃팅비_재정팀
```

#### 1.2 연도별 예산 설정

각 세목(BudgetDetail)에 대해 연도별로 다음을 설정합니다:

| 항목 | 설명 |
|------|------|
| 담당자 (Manager) | 해당 세목의 1차 결재자 |
| 예산금액 (Budget Amount) | 연도별 배정 예산 |
| 사용금액 (Used Amount) | 실시간 집계되는 사용 금액 |

#### 1.3 예산 현황 조회

- **사용률 표시**: 0-80% (녹색), 80-100% (주황), 100%+ (빨강)
- **잔액 표시**: 배정 예산 - 사용 금액
- **담당자 미지정 경고**: 빨간색으로 표시
- **엑셀 내보내기**: 예산 현황 보고서 생성

---

### 2. 지출결의서

#### 2.1 지출결의서 유형

| 유형 | 설명 | 접근 권한 |
|------|------|----------|
| **일반 지출결의서** | 위원회/사역팀 선택 → 세목 선택 방식 | 모든 사용자 |
| **간편 지출결의서** | 항목별로 세목 직접 선택 방식 | 관리자, 재정팀장, 회계, 행정간사 |

#### 2.2 지출결의서 구성

```
지출결의서 (Expense)
├── 기본 정보
│   ├── 위원회/사역팀
│   ├── 청구일자/지출일자
│   └── 청구인 정보
├── 항목 (ExpenseItem[])
│   ├── 예산(항/목/세목)
│   ├── 적요 (설명)
│   ├── 단가 × 수량 = 금액
│   └── (항목별 독립적인 예산 분류)
├── 은행 정보
│   ├── 은행명
│   ├── 계좌번호
│   └── 예금주
├── 첨부파일
└── 결재선
```

---

### 3. 결재 워크플로우

#### 3.1 3단계 결재 프로세스

```
1차 결재: 담당자 (세목별 지정된 Manager)
    ↓
2차 결재: 회계 (연도별 accountant 역할)
    ↓
3차 결재: 재정팀장 (연도별 finance_head 역할)
    ↓
최종 승인 (APPROVED_FINAL)
```

#### 3.2 결재선 자동 산출

- 첫 번째 항목의 세목 기준으로 결재선 자동 생성
- 담당자가 재정팀장인 경우 **전결 처리** (1차 자동 승인)
- 결재선 스냅샷 저장 (제출 시점 고정)

#### 3.3 결재 상태 흐름

```
DRAFT (작성중)
    ↓ [제출]
PENDING (결재 대기)
    ├─ [승인] → APPROVED_STEP_1 → APPROVED_STEP_2 → APPROVED_FINAL
    ├─ [반려] → REJECTED
    └─ [회수] → WITHDRAWN → [재제출] → PENDING
```

---

### 4. 지급 처리

#### 4.1 지급 상태

| 상태 | 설명 | 색상 |
|------|------|------|
| PENDING | 지급 대기 (기본값) | 노란색 |
| HOLD | 지급 보류 (사유 필수) | 주황색 |
| CANCELLED | 지급 취소 (사유 필수) | 빨간색 |
| COMPLETED | 지급 완료 | 녹색 |

#### 4.2 은행 연동 프로세스

```
┌─────────────────────────────────────────────────────────────┐
│                    지출결의서 시스템                           │
├─────────────────────────────────────────────────────────────┤
│  1. 지출결의서 작성 및 결재                                   │
│  2. 최종 승인 (APPROVED_FINAL)                               │
│  3. 지급 대기 상태 (PENDING)                                 │
│  4. 엑셀 내보내기                                            │
│     - 항, 목, 세목, 예금주, 은행, 계좌번호, 금액, 날짜        │
└──────────────────────┬──────────────────────────────────────┘
                       │
                       ▼ (엑셀 파일 다운로드)
┌─────────────────────────────────────────────────────────────┐
│                      은행 시스템                              │
├─────────────────────────────────────────────────────────────┤
│  5. 엑셀 파일 업로드                                         │
│  6. 일괄 이체 처리                                           │
└──────────────────────┬──────────────────────────────────────┘
                       │
                       ▼ (이체 완료 확인)
┌─────────────────────────────────────────────────────────────┐
│                    지출결의서 시스템                           │
├─────────────────────────────────────────────────────────────┤
│  7. 지급 완료 상태 변경 (COMPLETED)                          │
│  8. 예산 사용 금액 반영                                      │
│  9. 결산 데이터 확정                                         │
└─────────────────────────────────────────────────────────────┘
```

---

### 5. 감사 로그

모든 결재 액션은 ApprovalLog에 기록됩니다.

| 액션 | 설명 |
|------|------|
| SUBMIT | 지출결의서 제출 |
| APPROVE | 결재 승인 |
| REJECT | 결재 반려 |
| WITHDRAW | 작성자 회수 |
| RESUBMIT | 반려 후 재제출 |
| PAYMENT_COMPLETE | 지급 완료 처리 |
| PAYMENT_HOLD | 지급 보류 처리 |
| PAYMENT_CANCEL | 지급 취소 처리 |
| PAYMENT_REVERT | 지급 대기로 복원 |

## 데이터 모델 요약

### 핵심 테이블

```
예산 관련:
├── Committee (위원회)
├── Department (사역팀/부)
├── BudgetCategory (예산 항)
├── BudgetSubcategory (예산 목)
├── BudgetDetail (예산 세목)
└── BudgetDetailYear (연도별 세목 설정)

├── Expense (지출결의서)
├── ExpenseItem (지출 항목)
├── ExpenseAttachment (첨부파일)
├── ApprovalLine (결재선)
├── ApprovalStep (결재 단계)
└── ApprovalLog (감사 로그)

사용자 관련:
├── User (사용자)
├── UserYearRole (연도별 역할)
├── SavedBankAccount (저장된 계좌)
└── UserSignature (서명/도장)
```

## 현재 구현 상태

### ✅ 완료된 기능

| 분류 | 기능 |
|------|------|
| **예산 관리** | 5단계 계층 구조, 연도별 설정, 현황 조회, 엑셀 내보내기 |
| **지출결의서** | 일반/간편 양식, 항목별 예산 선택, 자동 금액 계산 |
| **결재** | 3단계 워크플로우, 자동 결재선 산출, 전결 처리, 서명/도장 |
| **지급 처리** | 4가지 상태 관리, 일괄 처리, 엑셀 내보내기 |
| **감사** | 모든 액션 로깅, 스냅샷 저장, 변경 이력 추적 |
| **기타** | 은행 계좌 저장, PDF/인쇄, PWA 지원 |

### 🎯 은행 연동 방식 (현재)

1. 최종 승인된 지출결의서 목록에서 대상 선택
2. "엑셀 내보내기" 버튼 클릭
3. 날짜 옵션 설정 (동일 날짜 적용 또는 기존 날짜 유지)
4. 엑셀 파일 다운로드 (지출재정 형식)
5. 은행 시스템에 엑셀 파일 수동 업로드
6. 은행에서 일괄 이체 처리
7. 시스템에서 지급 완료 상태 변경
