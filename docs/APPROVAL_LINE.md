# 결재선 설계 문서

## 개요

지출결의서 시스템의 결재선은 **예산 마스터(BudgetMaster)의 manager 필드**를 기반으로 자동 생성됩니다.

> **참고**: 현재 BudgetMaster 데이터는 **2025년도** 기준입니다.
> 2026년도 예산 데이터는 추후 일괄 업로드 또는 개별 등록 예정입니다.

---

## 1. 결재 프로세스

### 기본 3단계 결재
```
작성자 → 1차 팀장(Manager) → 2차 회계 → 3차 재정팀장 (최종)
```

### 자기결재 방지
작성자가 결재선에 포함된 경우 해당 단계를 자동으로 건너뜁니다.

---

## 5. 상태 전이

```
DRAFT (작성중)
   ↓ [제출]
PENDING (1차 결재 대기)
   ↓ [1차 승인]
APPROVED_STEP_1 (2차 결재 대기)
   ↓ [2차 승인]
APPROVED_STEP_2 (3차 결재 대기) - 3단계인 경우
   ↓ [최종 승인]
APPROVED_FINAL (최종 승인)

※ 어느 단계에서든 반려 시 → REJECTED
※ 작성자가 회수 시 → WITHDRAWN → 재제출 가능
```

## 6. 특수 케이스

### 복수 담당자가 있는 부서
- 공간사역팀: 신창국, 양찬승
- 이웃사랑팀: 신창국, 임한결
- 행정비: 신창국, 임대웅

→ 현재: 첫 번째 담당자가 1차 결재자로 지정
→ 향후: 복수 담당자 중 선택 또는 병렬 결재 기능 추가 가능

### 재정팀장 작성자
- 신창국이 작성한 경우 최종 결재자가 없음
- 담임목사 결재선 추가 필요 (현재 미구현)

## 8. 향후 계획

### 결재선 자동 생성 개선
- [ ] BudgetMaster의 manager 필드를 직접 조회하여 1차 결재자 결정
- [ ] 복수 담당자 처리 로직 추가
- [ ] 담당자 변경 시 실시간 반영

### 예산 데이터 연도별 관리
- 현재: 2025년도 예산 데이터
- 예정: 2026년도 예산 데이터 (일괄 업로드 또는 개별 등록)

### 예산 업로드 방식
1. **일괄 업로드**: Excel 파일로 전체 예산 데이터 업로드
2. **개별 등록**: 관리자 화면에서 항목별 등록/수정
