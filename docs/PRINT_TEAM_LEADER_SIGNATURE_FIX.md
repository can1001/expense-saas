# 지출결의서 프린트: 사역팀(부)장 서명/팀장명 출력 수정

## 요청사항

- **사역팀(부)장 아래 칸**: 서명(인) 표시/서명란
- **그 아래 칸**: 팀장 이름 표시

### 추가 요청(2026-01)

- 출력물에 하드코딩되어 있던 `신 창 국` 표기 위치에는 **팀장 이름을 표시**
- 기존 `재정팀장 전결` 영역은 **공란으로 표시**
- 이후 요구사항 변경: **재정팀장 이름 칸(우측 상단)은 재정팀장 이름 표시**, `재정팀장 전결` 영역은 **2개 로우를 1개(rowSpan=2)로 병합**

## 현재 상태(수정 전) 확인 결과

`components/print/PrintHeader.tsx` 기준으로,

- 좌측 1열에 `사역팀(부)장` 라벨은 있었지만,
- 바로 아래 2칸(행)이 `재정팀장 전결`(rowSpan=2)로 병합되어 있어
  **서명란/팀장명**을 넣을 공간 자체가 없었습니다.

즉, 출력물이 요구사항 구조(서명칸 → 이름칸)를 만족할 수 없는 테이블 구조였습니다.

## 수정 방향(핵심 아이디어)

1. **프린트 헤더 테이블 좌측 1열(사역팀장 영역) 행 구조를 변경**
   - `사역팀(부)장` 라벨 아래 2행을
     - 1행: **서명란(인)**
     - 1행 아래: **팀장 이름**
     로 재배치합니다.

2. **팀장 이름 데이터 소스**
   - 상세 화면은 이미 `/api/expenses/[id]/approval`을 호출해 결재선을 가져옵니다.
   - 해당 응답의 `approvalLine.steps`에서 `stepName === '팀장'`(또는 `approverTitle === '팀장'`)인 단계의 `approverName`을
     프린트용 팀장명으로 사용합니다.

## 적용된 변경 사항(코드)

### 1) 프린트 헤더 레이아웃 변경

- 파일: `components/print/PrintHeader.tsx`
- 변경:
  - 우측 상단의 `재정팀장` 이름 칸은 **재정팀장 이름(`financeManagerName`)**이 나오도록 변경했습니다.
  - 좌측 하단에 하드코딩되어 있던 `신 창 국` 위치는 **팀장 이름(`teamLeaderName`)**이 나오도록 변경했습니다.
  - `재정팀장 전결` 영역은 **공란**으로 출력하며, 시각적으로는 **2개 로우를 1개 셀(rowSpan=2)로 병합**했습니다.
  - 이름은 인쇄물 스타일을 맞추기 위해 글자 사이 공백을 넣어 표시합니다(예: `김흥래` → `김 흥 래`).

### 2) 프린트 컴포넌트에 팀장명 전달

- 파일: `components/PrintableExpense.tsx`
- 변경:
  - `teamLeaderName?: string | null` prop 추가
  - `PrintHeader`로 전달

### 3) 상세 페이지에서 팀장명 추출하여 프린트에 전달

- 파일: `app/expenses/[id]/page.tsx`
- 변경:
  - 이미 로드된 `approvalData`에서 팀장 단계의 `approverName`을 찾아
    `PrintableExpense`에 `teamLeaderName`으로 전달

## 테스트/검증 방법

1. 지출결의서 상세 화면(`/expenses/[id]`)에 진입
2. **프린트 버튼** 클릭
3. 인쇄 미리보기에서 아래를 확인
   - `사역팀(부)장` 바로 아래 칸에 **(인)** 이 표시되는지
   - 그 아래 칸에 **팀장 이름**이 표시되는지
     - (결재선이 아직 생성되지 않은 DRAFT 상태 등으로 `approvalLine`이 없으면, 팀장명은 공란으로 표시됩니다.)

## 참고

- 본 변경은 **프린트(HTML window.print)** 출력물의 헤더 테이블을 대상으로 합니다.
- 간편 지출결의서(`SimplePrintableExpense`) 출력물은 결재선/팀장 정보가 없어서 이번 수정 범위에 포함하지 않았습니다.
