## 1. 사용자 정보 DB 마이그레이션

### 배경
기존에는 `lib/users.ts`에 하드코딩된 사용자 정보를 사용했습니다. 유지보수성과 확장성을 위해 DB 기반으로 마이그레이션했습니다.

### 변경 내용

#### 새로운 Prisma 모델 (User)
```prisma
model User {
  id         String    @id @default(cuid())
  userid     String    @unique  // 로그인용 (청연홍길동)
  username   String              // 표시용 (홍길동)
  password   String?             // 추후 비밀번호 인증용
  role       UserRole  @default(user)
  department String?             // 소속 팀/부서
  isActive   Boolean   @default(true)
  createdAt  DateTime  @default(now())
  updatedAt  DateTime  @updatedAt
}
```

#### 새로운 API 엔드포인트
| 경로 | 메서드 | 설명 |
|------|--------|------|
| `/api/users` | GET | 사용자 목록 조회 (페이징, 검색, 필터) |
| `/api/users` | POST | 사용자 생성 |
| `/api/users/[id]` | GET | 사용자 상세 조회 |
| `/api/users/[id]` | PUT | 사용자 수정 |

| `/api/users/by-role/[role]` | GET | 역할별 사용자 조회 |

#### 새로운 서비스 레이어

## 3. 연도별 역할 관리 기능 추가 (UserYearRole)

### 배경
사용자의 역할이 연도별로 달라질 수 있어, 연도별 역할을 관리하는 기능을 추가했습니다.

### 변경 내용

#### 새로운 Prisma 모델 (UserYearRole)
```prisma
model UserYearRole {
  id        String   @id @default(cuid())
  userId    String
  year      Int
  role      UserRole
  user      User     @relation(fields: [userId], references: [id])
  createdAt DateTime @default(now())
  updatedAt DateTime @updatedAt

  @@unique([userId, year])
}
```

#### 새로운 API 엔드포인트
| 경로 | 메서드 | 설명 |
|------|--------|------|
| `/api/users/year-roles` | GET | 특정 연도/사용자의 역할 조회 |
| `/api/users/year-roles` | POST | 연도별 역할 설정 |
| `/api/users/year-roles` | DELETE | 연도별 역할 삭제 |

#### 새로운 서비스 함수
- `getEffectiveRole(userId, year)` - 해당 연도 유효 역할 조회
- `findUserWithEffectiveRole(userId, year)` - 유효 역할 포함 사용자 조회
- `findUsersByYearRole(year, role)` - 특정 연도/역할 사용자 목록
- `findAllUsersWithEffectiveRole(year)` - 전체 사용자 유효 역할 조회
- `setYearRole(userId, year, role)` - 연도별 역할 설정
- `deleteYearRole(userId, year)` - 연도별 역할 삭제
- `getYearRoles(userId)` - 사용자의 모든 연도별 역할 조회
- `canApproveByYear(userId, year)` - 해당 연도 결재 권한 확인

### 수정된 파일 (4개)
- `prisma/schema.prisma`
- `prisma/seed.ts`
- `lib/services/user-service.ts`
- `app/api/users/year-roles/route.ts` (신규)
