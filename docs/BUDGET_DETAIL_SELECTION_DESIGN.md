# 예산(세목) 선택 UI 설계 분석

## 현재 구현 방식

```
┌─────────────────────────────────────────────────────────────┐
│ 세부 항목                                      [+ 항목 추가] │
├─────────────────────────────────────────────────────────────┤
│  ┌─────────────────────────────────────────────────────┐    │
│  │ 예산(세목)                                          │    │
│  │ [교육교재비                              ▼]         │    │
│  │ 선택한 세목은 새 항목 추가 시 자동으로 적용됩니다      │    │
│  └─────────────────────────────────────────────────────┘    │
│                                                             │
│  ┌─ 항목 1 ─────────────────────────────────────────────┐   │
│  │ 예산(세목): [텍스트 입력]    적요: [텍스트 입력]      │   │
│  │ 단가: [1]                   인원(수량): [1]          │   │
│  └──────────────────────────────────────────────────────┘   │
└─────────────────────────────────────────────────────────────┘
```

**특징:**
- 상단에 공통 세목 선택 드롭다운
- 각 항목의 세목은 **텍스트 입력** 필드
- "빈 항목에 적용" 버튼으로 일괄 적용

---

## 이렇게 설계한 이유

### 1. 동일 세목 반복 입력 편의성
실제 지출결의서 작성 시 **동일한 세목으로 여러 항목**을 작성하는 경우가 많음:
```
예) 교육훈련위원회 - 교육자료비
  - 항목 1: 교육교재비 / 성경공부 교재 / 15,000원 × 10명
  - 항목 2: 교육교재비 / 워크북 / 8,000원 × 10명
  - 항목 3: 교육교재비 / 필기구 세트 / 3,000원 × 10명
```

### 2. 다른 세목 추가 가능성
하나의 지출결의서에 **다른 세목**을 포함해야 하는 경우도 있음:
```
예) 기획위원회 - 행사비
  - 항목 1: 식비_행사식비 / 점심 식사 / 10,000원 × 20명
  - 항목 2: 다과비 / 간식 / 5,000원 × 20명  ← 다른 세목
```

### 3. 예산 마스터에 없는 세목 직접 입력
일부 세목이 마스터 데이터에 없을 수 있어 **직접 입력**이 필요

---

## 대안 1: 각 항목에 드롭다운 배치

```
┌─ 항목 1 ────────────────────────────────────────────────┐
│ 예산(세목): [드롭다운 ▼]      적요: [텍스트 입력]       │
│ 단가: [1]                    인원(수량): [1]           │
└─────────────────────────────────────────────────────────┘
┌─ 항목 2 ────────────────────────────────────────────────┐
│ 예산(세목): [드롭다운 ▼]      적요: [텍스트 입력]       │
│ 단가: [1]                    인원(수량): [1]           │
└─────────────────────────────────────────────────────────┘
```

### 장점
- 각 항목에서 **직접** 세목 선택 가능
- 직관적인 UI
- 드롭다운으로 **오타 방지**

### 단점
- 동일 세목 반복 선택 시 **번거로움**
- 드롭다운에 없는 세목 입력 불가 (또는 "직접 입력" 옵션 필요)

### 구현 난이도: **낮음**
- ItemsSection에서 `detailOptions`를 각 항목의 드롭다운으로 전달하면 됨

---

## 대안 2: 드롭다운 + 직접 입력 혼합

```
┌─ 항목 1 ────────────────────────────────────────────────┐
│ 예산(세목): [드롭다운 ▼] 또는 [직접 입력]               │
│ 적요: [텍스트 입력]                                    │
│ 단가: [1]                    인원(수량): [1]           │
└─────────────────────────────────────────────────────────┘
```

### 장점
- 드롭다운과 직접 입력 **모두 가능**
- 유연성 확보

### 단점
- UI가 복잡해짐
- 사용자가 어떤 방식을 선택해야 할지 **혼란** 가능

### 구현 난이도: **중간**
- Combobox 또는 Autocomplete 컴포넌트 필요

---

## 대안 3: 현재 방식 유지 + 항목별 드롭다운 버튼 추가

```
┌─────────────────────────────────────────────────────────┐
│  [공통 세목 선택: 교육교재비 ▼]  [빈 항목에 적용]       │
└─────────────────────────────────────────────────────────┘

┌─ 항목 1 ────────────────────────────────────────────────┐
│ 예산(세목): [교육교재비        ] [▼]  적요: [...]       │
│            (텍스트 입력 가능)   (드롭다운)              │
└─────────────────────────────────────────────────────────┘
```

### 장점
- **기존 기능 유지** (텍스트 입력)
- 항목별로도 드롭다운 선택 가능
- 가장 **유연한** 방식

### 단점
- UI 요소가 많아짐

### 구현 난이도: **중간**

---

## 권장 방안

### 옵션 A: 대안 1 (각 항목에 드롭다운)
**가장 직관적**인 방식. 대부분의 경우 동일 목(目) 내의 세목만 사용하므로 드롭다운으로 충분.

```tsx
// ItemsSection.tsx 수정
<select
  {...register(`items.${index}.budgetDetail`)}
  className={SELECT_BASE}
>
  <option value="">세목 선택</option>
  {detailOptions.map((option) => (
    <option key={option} value={option}>{option}</option>
  ))}
</select>
```

### 옵션 B: 현재 방식 유지
다양한 세목을 사용해야 하는 **복잡한 지출결의서**에 적합.
다만, 상단 선택 영역의 목적을 더 명확히 안내 필요.

## 질문 사항

1. **실제 사용 패턴**: 하나의 지출결의서에 다른 세목을 혼합하는 경우가 얼마나 자주 있나요?
2. **예산 마스터 완성도**: 모든 세목이 마스터 데이터에 등록되어 있나요, 아니면 직접 입력이 필요한 경우가 있나요?
3. **사용자 선호**: 드롭다운 선택 vs 텍스트 입력 중 어떤 방식을 선호하시나요?

## 결론

| 방안 | 직관성 | 유연성 | 구현 난이도 |
|-----|-------|-------|-----------|
| 현재 방식 | 중 | 높음 | - |
| 대안 1 (항목별 드롭다운) | **높음** | 중 | 낮음 |
| 대안 2 (혼합) | 중 | **높음** | 중간 |
| 대안 3 (현재 + 버튼) | 중 | **높음** | 중간 |

**대부분의 사용 시나리오에서는 대안 1(각 항목에 드롭다운)이 가장 직관적**입니다.
다만, 다른 세목을 혼합해야 하는 경우가 많다면 현재 방식이 더 적합합니다.
